# 反応シミュレーション設計ドキュメント

## 0. ゲーム概要
- 女性向けワンシチュエーションエロゲ。
- マフィア幹部（イケメン）を緊縛拘束し、快楽堕ちさせる。
- Live2Dによる繊細な表情・身体表現がウリ。

## 1. 内部パラメータ
| パラメータ | 説明 | 値域 |
|---|---|---|
| `Arousal` | 快感の反応量 | `0.0` ～ `1.0` |
| `Edge Tension` | オーガズム欲 | `0.0` ～ `1.0` |
| `Resistance` | 抵抗感（高=拒絶、低=受容） | `0.0` ～ `1.0` |
| `Fatigue` | 消耗・疲弊 | `0.0` ～ `1.0` |
| `Drive` | 理性崩壊の総量 | `0.0` ～ `1.0` |
| `DriveBias` | 崩壊の質（`-`=トロトロ寄り、`+`=アヘ顔寄り） | `-1.0` ～ `1.0` |
| `NeedMotion` | 腰の反応・求める動き（内部蓄積） | `0.0` ～ `1.0` |
| `RejectMotion` | 拒絶や苛立ちを表現するランダム発火モーション。Resisに応じて発火率などが変化（内部蓄積） | `0.0` ～ `1.0` |
| `RejectHab` | 安定した刺激には慣れ、急な刺激がくるとRejectMotion発火率を上げるための関数（内部蓄積） | `0.0` ～ `1.0` |



## 2. 入力定義
### 2.1 MainIntensity（ピストン強度）
- スライダーで制御（`0.0` ～ `1.0`）。
- 手を離してもその値で停止・継続。
- `isActive` フラグで停止状態と区別。

### 2.2 SubA（チンコ刺激）
- クリック or バイブトグル（`ON/OFF`）。
- `Arousal` 増加 / `Resistance` 低下 / `DriveBias+` 方向 / `Drive` 増加。

### 2.3 SubB（乳首刺激）
- クリック or バイブトグル（`ON/OFF`）。
- `Arousal` 増加 / `Resistance` 増加（デフォルト） / `DriveBias-` 方向 / `Drive` 増加。
- `SubBResistanceChange` を負値に設定すれば状態ごとに低下に変更可能。

### 2.4 SubA + SubB 同時入力（実装仕様）
- SubA/B はそれぞれ独立して加算（単体ONと同時ONで特別処理は変わらない）。
- DriveBias のみ例外: 同時ON時は合算値の絶対値を**マイナス方向に強制** `(-|SubAGain + SubBGain|)`。

## 3. 入力帯（Band）判定
- 状態ごとに `TolLow` / `TolHigh` が変わる。

| 入力帯 | 条件 |
|---|---|
| `Stop` | `isActive == false` |
| `Below` | `MainIntensity < TolLow` |
| `Within` | `TolLow <= MainIntensity <= TolHigh` |
| `Above` | `MainIntensity > TolHigh` |


## 4. 状態定義と遷移
### 4.1 状態ごとの閾値
| 状態 | TolLow | TolHigh | 備考 |
|---|---:|---:|---|
| ① `Guarded` | 0.30 | 0.50 | 狭い、中間のみ許容 |
| ② `Defensive` | 0.20 | 0.40 | 刺激に敏感で嫌がる範囲が広い |
| ③ `Overridden` | 0.30 | 0.70 | わりとなんでも `Within` |
| ④ `FrustratedCraving` | 0.25 | 0.55 | 弱い刺激が `Within`（焦らし感） |
| ⑤ `Acclimating` | 0.30 | 0.70 | ③と同程度 |
| ⑥ `Surrendered` | 0.20 | 0.80 | ほぼ全域 `Within` |
| ⑦ `BrokenDown` | 0.15 | 0.85 | ほぼ全域 `Within`

### 4.2 各状態の説明と遷移
#### ① Guarded（初期状態）
- 拘束されて冷や汗をかきながらも、心を閉ざしてニヤッとしている。
- 初期Resistance: `0.5`（中間値）
- 遷移:
  - `Above` 継続 -> ② `Defensive` (抵抗感が高まり防衛)
  - `Resistance` 閾値以上 -> ② `Defensive`（抵抗感が高まると防衛）
  - `Within` 安定継続 + `Arousal` 閾値以上 -> ⑤ `Acclimating` (気持ちよくなって馴化へ)
  - `Below` 継続 + `Arousal` 閾値以上 -> ⑤ `Acclimating`（焦らし的なアプローチでも馴化へ）

#### ② Defensive
- 乱暴に扱われて心を閉ざした不機嫌状態。
- 遷移:
- `Above` 執拗継続 -> ③ `Overridden`
- `Within` 安定継続 -> ① `Guarded`
- `Below` 執拗継続 -> ④ `FrustratedCraving`
##### ★変更希望
- Resistance閾値以下で①遷移に変更。wihin継続での遷移は撤廃（抵抗感下がって戻る）
- Fatigue閾値以上かつDrive閾値以上で③遷移に変更。Above継続での遷移は撤廃（疲れと興奮マックスで屈服）
- メイン停止またはBelowが一定時間以上かつDrive閾値以上で④遷移に。（興奮してるのに穏やかな刺激しかこないのでお預け状態に）

#### ③ Overridden
- 強い快感で押し切られて射精する。
- 遷移:
- **突入後2回以上射精** かつ `Fatigue` 閾値以上 -> `End_A`（グッタリエンド）
- `Arousal` が閾値（`TransitionOverriddenExitArousal`）以下 -> ② `Defensive`（射精後など興奮が収まって防衛に戻る）
- `Within` 安定継続 -> ⑤ `Acclimating`（受け入れモードへ移行）
- `Below` 執拗継続 -> ④ `FrustratedCraving`
##### ★変更希望
- Fatigue閾値以下かつArousal閾値以下で②遷移に変更。（落ち着いて防衛に戻る）
- Fatigue閾値以下かつArousal閾値以上で⑤遷移に変更。wihinでの遷移は撤廃。（気持ちよくなって受け入れモードへ移行）
- Belowが一定時間以上かつDrive閾値以上で④遷移に（興奮してるのにぬるい刺激でお預け状態へ）


#### ④ FrustratedCraving（お預け状態）
- 強い刺激の後に急に弱くなり、精神は閉ざしているのに体が求めてしまう困惑状態。
- 遷移:
- `Within` 継続 -> ⑤ `Acclimating`（焦らし後に馴化へ戻る）
- `Below` 執拗継続 + `Arousal` 閾値以上 -> ⑥ `Surrendered`（物足りなさで降参）
- `Above` 執拗継続 -> ③ `Overridden`（強引に押し切られる）

##### ★変更希望
- within、Below、Above継続時間のみのトリガー遷移撤廃。
- Fatigue閾値以上で⑤に遷移（疲労して射精モードへ）
- Belowが一定時間以上かつDriveまたはArousal閾値以上で⑥遷移に（優しい刺激で降参）
- Resistance閾値以下かつArousal閾値以上で⑤遷移に（防衛がとけて気持ちよくなると馴化へ）

#### ⑤ Acclimating（馴化）
- 心理的ガードが下がり、身体が快楽を受け入れていく状態。
- 遷移:
- `Within` 安定継続 + `Arousal` 閾値以上 -> ⑥ `Surrendered`
- `Below` 執拗継続 + `Arousal` 閾値以上 -> ⑥ `Surrendered`（物足りなさで降参）
- `Drive` 閾値超え + `Above` 執拗継続 + `Arousal` 閾値以上 -> ⑦ `BrokenDown`（DriveBias `+|DriveBias|` 変換）

#### ⑥ Surrendered（通常快楽落ち）
- 許容値が広くなり、なんでも気持ちよくなる状態。
- 遷移:
- **突入後2回以上射精** かつ `Fatigue` 閾値以上 -> `End_B`（快楽落ちエンド）
- `Drive` 閾値超え + `Below` 執拗継続 + `Arousal` 閾値以上 -> ⑦ `BrokenDown`（DriveBias `-|DriveBias|` 変換）

#### ⑦ BrokenDown（理性崩壊・サービスタイム）
- 理性が完全に崩壊し、弱い刺激でも強い刺激でもイってしまう状態。
- ⑦突入時の処理:
- 遷移元Bandに応じて `DriveBias` を変換してからモードをロック（サービスタイム中は変更しない）
  - `Below` 継続で突入 -> `DriveBias = -|DriveBias|` -> トロトロモード確定
  - `Above` 継続で突入 -> `DriveBias = +|DriveBias|` -> アヘ顔モード確定
  - ⑦直行は現在の `DriveBias` をそのまま使用
- `TolLow=0.05` / `TolHigh=0.95` に設定（ほぼ全刺激が `Within` に入る）
- 射精閾値・射精後Arousalは ③⑥ と同一設定（イキやすさは `ArousalGain` の高さで表現）
- `Fatigue` 蓄積速度を2倍に設定（`FatigueMultiplier=2.0`）
- `Drive` などの内部パラメータは他の状態と同様に増減し続ける（強い興奮状態なのでDriveは高めに設定推奨）
- モード分岐（突入時にロック）:
- `DriveBias >= 0` -> アヘ顔モード -> `End_C_Overload`
- `DriveBias < 0` -> トロトロモード -> `End_C_White`
- 遷移:
- **突入後3回以上射精** かつ `Fatigue` 閾値以上 -> `End_C_White` or `End_C_Overload`

## 5. エンディング一覧
| エンド | 条件 | 演出 |
|---|---|---|
| `End_A` | ③突入後2回以上射精 + `Fatigue` 閾値以上 | 潮吹きグッタリスチル |
| `End_B` | ⑥突入後2回以上射精 + `Fatigue` 閾値以上 | 快楽落ちスチル |
| `End_C_White` | ⑦突入後3回以上射精 + トロトロ + `Fatigue` 閾値以上 | とろけ落ちスチル |

## 6. パラメータ更新ルール
### 6.1 共通ルールのざっくり解説
| 観点 | ざっくり要点 |
|---|---|
| `Arousal` | 刺激で上がり、停止で下がる。射精時は `OrgasmArousalResetTo` 以下にクランプ（完全リセットしない）。 |
| `Resistance` | 基本は削れるが、`Above` では反発で上がる。 |
| `Fatigue` | `Within/Above` で溜まり、停止で回復する。射精時に `OrgasmFatigueGain` 分だけ即時増加。 |
| `Drive` | `Below/Above` を一定時間続けると増え、`Stop` で下がる。`Within` は**保持**（`DriveDecayWithin=0`）。SubA/B入力でも増加。 |
| `DriveBias` | 刺激文脈と Sub入力で `-1 ～ +1` のどちらかに寄る。⑦突入時に遷移元Bandで符号変換される。 |
| `NeedMotion` | `Below` で増加、`Within`/`Above` で減少。興奮強い状態（③⑥⑦）は `Above` のみ減少・`Within` はキープ。`Stop` はArousal高い or Sub有りで増加。 |

### 6.2 状態別係数のざっくり解説
| 観点 | ざっくり要点 |
|---|---|
| 全体傾向 | 更新係数の多くは全状態で共通。差分は一部の要所係数に集中。 |
| 入力帯の幅 | `TolLow/TolHigh` は状態ごとに異なり、`Within` 判定の入りやすさが変わる。 |
| Driveの溜まりやすさ | `Overridden / FrustratedCraving / Acclimating` は `DriveGainBelow/Above=0.12` で高い。`BrokenDown` は強い興奮状態なので高め推奨。 |
| 疲労の重さ | `FatigueMultiplier` は `BrokenDown` のみ `2.0`。その他は `1.0`。 |
| NeedMotion | `Below` 増加 / `Within` 減少（興奮強い状態はキープ）/ `Above` 減少 / `Stop` 条件付き増加（係数はConfig）。 |

### 6.3 共通ルール（式）
`dt = Time.deltaTime`。最終的に `Arousal/Resistance/Fatigue/Drive/NeedMotion` は `[0,1]`、`DriveBias` は `[-1,1]` クランプ。

- Band更新（毎フレーム）
  - `Stop`
    - `Arousal -= ArousalDecayStop * dt`
    - `Resistance -= ResistanceDecayStop * dt`
    - `Fatigue -= FatigueDecayStop * dt`
    - `Drive -= DriveDecayStop * dt`
    - `DriveBias -> 0`（`DriveBiasDecayStop * dt`）
    - `above/below/within` と `driveRampTimer` をリセット
  - `Below`
    - `Arousal += ArousalGainBelow * dt`
    - `Resistance -= ResistanceDecayBelow * dt`
    - `Fatigue -= 0.005 * dt`（固定）
    - `DriveBias -> -1`（`DriveBiasShiftBelow`）
    - `driveRampTimer += dt`、`DriveTimeBeforeGain` 到達後 `Drive += DriveGainBelow * dt`
  - `Within`
    - `Arousal += ArousalGainWithin * dt`
    - `Resistance -= ResistanceDecayWithin * dt`
    - `Fatigue += FatigueGainWithin * dt * FatigueMultiplier`
    - `Drive -= DriveDecayWithin * dt`
    - `DriveBias -> 0`（`DriveBiasDecayWithin`）
    - `driveRampTimer` リセット
  - `Above`
    - `Arousal += ArousalGainAbove * dt`
    - `Resistance += ResistanceRiseAbove * dt`
    - `Fatigue += FatigueGainAbove * dt * FatigueMultiplier`
    - `DriveBias -> +1`（`DriveBiasShiftAbove`）
    - `driveRampTimer += dt`、`DriveTimeBeforeGain` 到達後 `Drive += DriveGainAbove * dt`

- Sub入力追加（係数はすべてConfigから、状態ごとにOverrideSub可能）
  - `SubA ON`: `Drive += SubADriveGain*dt`, `Arousal += SubAArousalGain*dt`, `Resistance += SubAResistanceChange*dt`（デフォルト負値=低下）
  - `SubB ON`: `Drive += SubBDriveGain*dt`, `Arousal += SubBArousalGain*dt`, `Resistance += SubBResistanceChange*dt`（デフォルト正値=増加）
  - DriveBias: `biasDelta = (SubAがONなら+SubADriveBiasGain) + (SubBがONなら+SubBDriveBiasGain)`
    - 同時ON時のみ `biasDelta = -|biasDelta|`（絶対値をマイナスに強制）
  - `SubA + SubB 同時ON` かつ `SubBothExtraArousal > 0`: `Arousal += SubBothExtraArousal*dt` を追加

- 射精時処理
  - `Arousal = Min(Arousal, OrgasmArousalResetTo)`（完全リセットせず部分的に下げる）
  - `Fatigue += OrgasmFatigueGain`（ぐっと増加）

- NeedMotion更新（新仕様）
  - `isHighExcitement = state は Overridden / Surrendered / BrokenDown のいずれか`
  - `Stop` 帯:
    - `Arousal >= NeedMotionArousalThreshold` または Sub有り -> `NeedMotion += NeedMotionStopGain*dt`
    - それ以外 -> `NeedMotion -= NeedMotionStopDecay*dt`
  - `Below` 帯: `NeedMotion += NeedMotionBelowGain*dt`（全状態共通）
  - `Within` 帯:
    - 通常状態かつ `NeedMotionWithinGain > 0` -> `NeedMotion += NeedMotionWithinGain*dt`（増加・特殊状態設定時）
    - 通常状態かつ `NeedMotionWithinGain == 0` -> `NeedMotion -= NeedMotionWithinDecay*dt`（デフォルト減少）
    - 興奮強い状態 -> 変化なし（キープ）
  - `Above` 帯: `NeedMotion -= NeedMotionAboveDecay*dt`（全状態共通）

| `EndCOrgasmCount` | 3 | ⑦でEnd_Cに必要な射精回数（状態上書き不可） |

## 7. 出力パラメータ（Live2D連携用）
- 内部パラメータを直接 Live2D に渡さず、中間の表現パラメータに変換。
- すべて `0.0` ～ `1.0` に正規化。
- Animator/Cubism 側では `AnimationBridge` で `SmoothDamp` 補間をかける設計。
- 出力変換係数（`OutputWeights`）は Inspector で調整可能。

### 7.1 内部パラメータ → 出力パラメータ変換式（実装）
`driveBiasMinus = Max(0, -DriveBias)`

- `BodyTension = Clamp01(Resistance * BodyTensionResistance + Arousal * BodyTensionArousal)`
- `BodyYield   = Clamp01((1 - Resistance) * 0.6 + Arousal * 0.4)`
- `BreathDepth = Clamp01(Arousal * BreathDepthArousal + Fatigue * BreathDepthFatigue)`
- `FaceHeat    = Clamp01(Arousal * FaceHeatArousal + driveBiasMinus * FaceHeatDriveBias)`
- `EyeFocus    = Clamp01((1 - Drive) * EyeFocusDrive + Resistance * EyeFocusResistance)`
- `ControlMask = Clamp01(Resistance * 0.5 + (1 - Drive) * 0.5)`
- `NeedMotion  = Clamp01(NeedMotion * NeedMotionAccumulated + Drive * NeedMotionDrive + driveBiasMinus * NeedMotionDriveBias)`
- `PeakDrive   = Drive`
- `Aftershock`:
  - 射精フレームで `1.0`
  - それ以外は `MoveTowards(0, AftershockDecaySpeed * dt)` で減衰

| 出力値 | 主な対応内部パラメータ | 割当先（想定） |
|---|---|---|
| `BodyTension` | `Resistance`, `Arousal` | 体幹・肩の緊張 |
| `BodyYield` | `1-Resistance`, `Arousal` | 腰の脱力・反応 |
| `BreathDepth` | `Arousal`, `Fatigue` | 呼吸の深さ・乱れ |
| `FaceHeat` | `Arousal`, `DriveBias-` | 顔の赤み・火照り |
| `EyeFocus` | `1-Drive`, `Resistance` | 目の焦点（理性の残量） |
| `ControlMask` | `Resistance`, `1-Drive` | 感情抑制の度合い |
| `NeedMotion` | `NeedMotion(内部蓄積)`, `Drive`, `DriveBias-` | 腰の反応・求める動き |
| `PeakDrive` | `Drive` | 崩壊演出の強度 |
| `Aftershock` | `Fatigue`（射精直後） | 余韻の震え |

## 8. 実装クラス構成（推奨）
```text
SimState.cs              // enum SimState（7状態 + End定義）
SimParameters.cs         // 内部パラメータのデータクラス
SimStateConfig.cs        // 状態ごとのTolLow/TolHighなど（ScriptableObject）
InputHandler.cs          // MainIntensity / SubA / SubB の入力受け取り
BandEvaluator.cs         // Below / Within / Above 判定
ParameterUpdater.cs      // パラメータ毎フレーム更新
StateResolver.cs         // 状態遷移判定
EndJudge.cs              // エンド条件判定
OutputDriver.cs          // SimulationOutput計算（Live2D連携ポイント）
SimulationOutput.cs      // 出力DTO構造体
SimulationManager.cs     // MonoBehaviour、上記を統括
```

## 10. Unity側で手動が必要な作業
1. Animator Controller の作成、各状態のアニメーションクリップ設定。
2. Live2D Cubism SDK のインポートとモデル設定。
3. `OutputDriver` の各出力値を Animator Parameter / Cubism Parameter に接続。
4. ScriptableObject（`SimStateConfig`）を Inspector で各状態分作成・割当。
5. スライダーUIの `OnValueChanged` を `InputHandler.SetMainIntensity` に接続。
6. SubA / SubB トグルボタンの `OnClick` を `InputHandler.SetSubA` / `SetSubB` に接続。
